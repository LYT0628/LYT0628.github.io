hexo博客使用

  • 修改文章（博客等）在github更新的步骤

    1. 在本地blog/source/posts 中直接修改文章

    2. 在博客根目录打开git bash here

    3. ctrl+c 接 hexo s命令，重启hexo服务

    4. 在 git bash here 中执行hexo clean &&hexo g hexo s接hexo d指令

    5. 等待片刻，刷新页面

      ps：可以自行测试：

      1. 在blog/source/posts 中修改
        1. 本地博客自动更新
        2. GitHub中文档不更新
      2. ctrl+c 接 hexo s命令后，GitHub文档仍然不更改
      3. hexo clean &&hexo g&& hexo s接hexo d指令后，github网页更改
  • 新建文档的方式
    • 直接在本地blog/source/posts 中插入 .md文件
    • 使用hexo的命令
      • hexo提供三种布局：post，draft和page
        • post文档，保存在source/_posts文件夹中。文章布局，顾名思义就是你已发布的文章，会在博客网站里显示
        • draft文档，保存在source/_drafts文件夹中。草稿布局，顾名思义就是你还没写完不想发布的文章，默认不会在博客网站里显示。
        • pege，Hexo会创建一个目录，并在其中新建index.md文件。页面布局，顾名思义就是用来DIY我们博客页面的。
      • hexo new "Mytext"不指定布局类型，文档为默认布局
      • hexo new post "Mytext1",指定为post布局
      • hexo new draft "Mytext2",指定为draft布局
        • hexo publish post "Mytext2"，移动草稿到posts
      • hexo new page "Mytext3",指定为page布局
  • 文档的书写格式

    • 格式部分（文档头）

      • 形如

      • 有以下几种格式

        • title：hello world #标题，默认文章名

        • data：2022-10-22 #文章发布的时间，有默认

        • updata：2022-10-22#文章更新时间，有默认

        • comments： true #评论功能,默认true，可改为false

        • 标签与分类

          • 分类是个人博客下的hexo博客
          • 文章拥有两个标签：hexo和博客
          • 它属于三个分类：日常 > 生活，日常 > 随想，日记
          • 其中生活、随想为日常的子分类，日常和日记为同级分类
          • ！分类（categories）和标签（tags）都需要自行创建页面后使用（见下文）
  • 修改博客主题：

    • hexo博客根目录的_config.yml文件叫做站点配置文件

    • hexo主题中心

      1.在hexo主题中心找到喜欢的主题

      2.下载安装包，解压主题到blog/source/themes中

      3.将站点配置文件themes值修改为主题文件夹名

      • 主题文件夹也有一个.yml文件，叫主题配置文件，用来修改主题
    • 以NexT为例的主题配置文件具体修改（仅显示重要修改，+表示必须修改，-表示非必须修改）
      • -风格样式设置（/**/表示笔者的注释，#是NexT原代码的注释，//表示笔者也不清楚，后补，可自行百度）
      • +菜单设置

        • 以创建tags为例（要什么页面看自己需求）

          1. 在blog根目录页面打开git bash here，执行hexo new page tags，若创建categories就是hexo new page categories,其余同理

          2. 在source/tags/index.md文件中，文档头中书写type: "tags"/注意：冒号后有空格，全文其余部分都一样/

          3. 在主题配置文件中，找到（ctrl+f）menu部分，将下方对应部分的‘#’删除

            到这，页面就创建完成啦
